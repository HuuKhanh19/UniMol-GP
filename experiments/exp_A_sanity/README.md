# Experiment A — Sanity check: EGGROLL vs Gradient Descent trên Linear head

## Mục tiêu

Verify rằng implementation EGGROLL của chúng ta hoạt động đúng bằng cách so sánh
với gradient descent trên một bài toán "control": train một `nn.Linear(512 -> 1)`
trên top of **frozen** Unimol-v1 embeddings cho ESOL.

Vì model chỉ có 513 params và task có closed-form solution (ridge regression),
**cả ba phương pháp dưới đây phải cho metric gần nhau**:

| Phương pháp | Vai trò |
|---|---|
| Closed-form ridge | Reference (analytical optimum) |
| Gradient descent (Adam) | Baseline ML standard |
| EGGROLL (Adam) | Cái chúng ta đang verify |

Nếu EGGROLL match được GD trên Experiment A → implementation đúng → chuyển sang
Experiment B (fine-tune full encoder).

## Kỳ vọng kết quả

Trên ESOL (1128 mols, ~1015 train + ~113 val ở seed 42), với frozen Unimol-v1
embeddings, mức performance baseline điển hình:

| Method | val RMSE | val R² | val Pearson |
|---|---|---|---|
| Closed-form ridge | ~0.65–0.75 | ~0.78–0.85 | ~0.89–0.92 |
| Gradient descent | ~0.65–0.75 | ~0.78–0.85 | ~0.89–0.92 |
| **EGGROLL (verify target)** | **trong 5% của GD** | **trong 0.02 của GD** | **trong 0.02 của GD** |

`03_compare.py` sẽ tự động in verdict (PASS / MARGINAL / FAIL).

## Tham số EGGROLL — giải thích từng cái

| Param | Default | Ý nghĩa |
|---|---|---|
| `--sigma` | 0.05 | Độ lớn nhiễu. Quá lớn → fitness noisy, quá nhỏ → exploration kém |
| `--pop-size` | 256 | Số candidate đánh giá mỗi step. Càng lớn càng chính xác gradient, càng tốn compute. Phải chẵn (do antithetic) |
| `--lr` | 0.01 | Learning rate của outer Adam optimizer áp dụng EGGROLL gradient |
| `--epochs` | 500 | Số update steps |
| `--batch-size` | 64 | Mini-batch dùng để tính fitness mỗi step (giống batch của GD) |
| `--fitness-shaping` | zscore | `zscore` ổn định nhất. `rank` robust hơn với outlier. `raw` không khuyến nghị |

## Debug guide nếu PASS không đạt

### Nếu EGGROLL stuck (fitness không tăng):
1. `sigma` quá nhỏ → tăng lên 0.1
2. `lr` quá nhỏ → tăng lên 0.05
3. Check sign: noise should be normal(0, 1), gradient negated cho minimization
4. Verify antithetic: nửa đầu noise tensor + nửa sau = 0

### Nếu EGGROLL divergent (loss bùng nổ):
1. `sigma` quá lớn → giảm xuống 0.01
2. `lr` quá lớn → giảm xuống 0.001
3. Thử `--fitness-shaping rank` thay vì zscore

### Nếu EGGROLL hội tụ chậm hơn nhiều so với GD:
1. Bình thường — ES dùng nhiều sample hơn để estimate gradient
2. Tăng `pop_size` lên 512 hoặc 1024
3. Hoặc đơn giản train nhiều `epochs` hơn
